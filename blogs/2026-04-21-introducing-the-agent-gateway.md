<img alt="Introducing the Agent Gateway" src="https://storage.ghost.io/c/04/80/04808bfa-0485-4139-98cc-e3f82ab7f009/content/images/2026/04/Thumbnail.png" /><p>Portkey&apos;s mission has always been to help teams run AI in production reliably, with full governance and visibility. We spent the last few years building that stack for LLM calls. Thousands of teams run on it today, processing over 1 trillion tokens every day.</p><p>Since then, production AI has changed. Enterprises are no longer shipping simple AI apps. They are building agents that run autonomously, chain tool calls, consume MCP servers, and operate over longer timescales with real consequences.</p><p>Every enterprise that ships an agent hits the same wall fast: who can call this, what does it cost, what did it do, and who is responsible when something goes wrong. Every team patches this together themselves. Today we are shipping the infrastructure that solves this.</p><h2 id="agent-gateway">Agent Gateway</h2><p>Portkey&apos;s Agent Gateway is the production layer between your agents and the world.</p><p>Register any agent, get a governed Portkey endpoint, and every call flows through a single production stack from that point on. Your agent code doesn&apos;t change. Your clients don&apos;t change. You get one place to see what&apos;s happening, control who can do what, and enforce policy across your entire org.</p><figure class="kg-card kg-image-card"><img alt="Introducing the Agent Gateway" class="kg-image" height="1199" src="https://storage.ghost.io/c/04/80/04808bfa-0485-4139-98cc-e3f82ab7f009/content/images/2026/04/image-31.png" width="2000" /></figure><p>Here&apos;s what the Agent Gateway brings you:</p><ul><li><strong>Governance and access control</strong> Scoped per agent, per team, and per user. Budget and usage limits set once, enforced everywhere. Policy changes apply instantly across the org, no redeployment.</li><li><strong>Full observability</strong> Complete traces across every agent run, including every MCP call your agents make. 40+ metrics out of the box. If something went wrong, you have the full record.</li><li><strong>Reliability built in</strong> Automatic fallbacks and load balancing at the gateway layer. Provider outages get resolved before they reach your agents.</li><li><strong>Security and guardrails</strong> PII detection, PHI filtering, and content moderation. 50+ guardrails available without any custom setup.</li><li><strong>One auth model</strong> Consistent across every agent regardless of how it was built. API keys, JWT, Basic Auth, MCP auth spec, fine-grained RBAC at every level.</li><li><strong>Agent Registry</strong> As deployments grow, the first thing that breaks is visibility. Which agents exist, who owns them, who has access. Registry gives every agent a permanent home: its own governed endpoint, its own access controls, its own Skills and Capabilities definition. A service catalog built specifically for agents, with auth and RBAC native to every entry.</li></ul><h2 id="getting-started">Getting Started</h2><p>Agent Gateway is available in beta today.</p><p>Just as Portkey&apos;s AI Gateway became the production layer for LLM calls, Agent Gateway is the production layer for the agents built on top of them.
